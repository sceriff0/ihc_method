Purpose

Validate the FlowPath IHC readout against the pathologist. Two independent checks:

  1. Neoplastic cellularity — IHC tumour fraction inside each annotation vs the pathologist’s neoplastic_data estimate. Cell membership in the annotation is recomputed from the raw geojson with sf (not the CSV Out_of_annotation flag), and reported both per-annotation and over the dissolved union.
  2. Immune infiltrate — IHC CD45/lineage fractions inside the tumour region vs the clinical HOT score / Immuno-phenotype.
  3. Biological-validity QC — do the IHC-derived metrics behave the way real biology predicts against the wider clinical table? Three cross-checks: peripheral-blood immunity (lymphocytes / NLR / PNI), induction-treatment RESPONSE, and survival (OS, PFS). A genuine immune readout should track peripheral lymphocytes, better response, and longer survival.

Tumour cell = phenotype_clean containing “Tumor” (PANCK+Tumor, VIM+Tumor).

Checks 1–2 use the annotation region (sf); check 3 uses whole-slide IHC immune metrics so it covers every imaged patient, not only the handful with pathologist annotations.

plot_concordance <- function(df, subtitle) {
  st <- paired_spearman(df$tumor_over_inside, df$path_frac)
  ggplot(df, aes(tumor_over_inside, path_frac)) +
    geom_abline(slope = 1, linetype = "dashed", colour = "red") +
    geom_point(alpha = 0.7, size = 2) +
    coord_equal(xlim = c(0, 1), ylim = c(0, 1)) +
    annotate("text", x = 0, y = 1, hjust = 0, vjust = 1,
             label = sprintf("Spearman rho = %.2f (p = %.3g, n = %d)",
                             st$rho, st$p, st$n)) +
    labs(title = "Neoplastic cellularity: IHC vs pathologist", subtitle = subtitle,
         x = "IHC tumour fraction inside annotation",
         y = "Pathologist tumour fraction") +
    theme_classic(base_size = 12)
}

Survival — does the IHC immune readout carry prognostic signal?

Kaplan–Meier by median split of the IHC immune score, plus a univariable Cox HR on the continuous score, for OS and PFS.

Assumption to verify: the time columns are OS / PFS (numeric months) and the event columns are FOLLOW UP (death) / PERSISTENCE OR RELAPSE? (progression). coerce_event() maps common encodings (yes/no, alive/dead, 1/0) to 0/1 — adjust surv_cfg if your coding differs.

surv_cfg <- list(
  OS  = list(time = "OS",  event = "FOLLOW UP"),
  PFS = list(time = "PFS", event = "PERSISTENCE OR RELAPSE?")
)

coerce_event <- function(x) {
  if (is.numeric(x)) return(as.integer(x > 0))
  v <- tolower(trimws(as.character(x)))
  dplyr::case_when(
    v %in% c("1","yes","y","dead","death","deceased","event","relapse",
             "progression","progressed","true") ~ 1L,
    v %in% c("0","no","n","alive","ned","censored","false") ~ 0L,
    TRUE ~ NA_integer_
  )
}

run_survival <- function(cfg, label) {
  if (!requireNamespace("survival", quietly = TRUE)) {
    message("survival package not installed — skipping ", label); return(invisible())
  }
  if (!all(c(cfg$time, cfg$event) %in% names(ihc_clin))) {
    message("missing ", cfg$time, "/", cfg$event, " — skipping ", label); return(invisible())
  }
  d <- ihc_clin |>
    transmute(ihc_immune_score,
              .time  = suppressWarnings(as.numeric(.data[[cfg$time]])),
              .event = coerce_event(.data[[cfg$event]])) |>
    filter(is.finite(.time), !is.na(.event), is.finite(ihc_immune_score))
  if (nrow(d) < 5 || dplyr::n_distinct(d$.event) < 2) {
    message("too few informative rows for ", label, " (n = ", nrow(d), ")"); return(invisible())
  }
  d <- d |> mutate(immune_group = if_else(ihc_immune_score >= median(ihc_immune_score),
                                          "IHC-immune high", "IHC-immune low"))
  fit  <- survival::survfit(survival::Surv(.time, .event) ~ immune_group, data = d)
  cox  <- survival::coxph(survival::Surv(.time, .event) ~ ihc_immune_score, data = d)
  hr   <- exp(stats::coef(cox))[1]; pcox <- summary(cox)$coefficients[1, "Pr(>|z|)"]
  cat(sprintf("%s: Cox HR per unit IHC immune score = %.2f (p = %.3g, n = %d)\n",
              label, hr, pcox, nrow(d)))
  if (requireNamespace("survminer", quietly = TRUE)) {
    print(survminer::ggsurvplot(fit, data = d, pval = TRUE, risk.table = TRUE,
                                title = paste(label, "by IHC immune score (median split)"),
                                legend.title = "")$plot)
  } else {
    plot(fit, col = c("firebrick", "steelblue"), xlab = "time", ylab = "survival",
         main = paste(label, "by IHC immune score"))
    legend("topright", levels(factor(d$immune_group)),
           col = c("firebrick", "steelblue"), lty = 1, bty = "n")
  }
}

Notes

  • Cohorts are small (the pathologist scored a handful of slides); Spearman is used throughout and every correlation reports its n. Read associations as descriptive, not inferential.
  • Per-annotation vs union: per-annotation tests each scored region directly; the union collapses a patient to one tumour-content value (pathologist reference = mean of their annotations).
  • Membership is sf::st_within on the raw geojson — a cell counts as inside only if its centroid falls in the polygon, independent of the CSV flag.
  • Biological-validity QC (blood / response / survival) is exploratory and underpowered at this cohort size — it checks the direction the IHC immune readout points (with peripheral lymphocytes, with response, with survival), not statistical significance. Confirm the surv_cfg time/event columns match your data coding before trusting the KM/Cox output.
